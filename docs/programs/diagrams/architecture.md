## 3. Cross-Program Invocation (CPI) Flow

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          CPI DEPENDENCY MATRIX                                  │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  FROM ↓ / TO →    │ Registry │ Oracle │ Governance │ Energy Token │ Trading   │
│  ─────────────────┼──────────┼────────┼────────────┼──────────────┼───────────│
│  Registry         │    -     │   -    │     -      │    CPI ✓     │    -      │
│  Oracle           │  CPI ✓   │   -    │     -      │      -       │  CPI ✓    │
│  Governance       │  READ ✓  │   -    │     -      │      -       │    -      │
│  Energy Token     │    -     │   -    │     -      │      -       │    -      │
│  Trading          │  READ ✓  │ READ ✓ │   READ ✓   │    CPI ✓     │    -      │
│                                                                                 │
│  ✓ CPI = Cross-Program Invocation (write)                                      │
│  ✓ READ = Account data read (no CPI)                                           │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

DETAILED CPI FLOWS:

1. Oracle → Registry (update_meter_reading)
   ┌────────────┐                    ┌────────────┐
   │   Oracle   │ ──CPI──────────▶  │  Registry  │
   │            │   update_meter_   │            │
   │ Validates  │   reading()       │ Updates    │
   │ reading    │                   │ MeterAcct  │
   └────────────┘                   └────────────┘

2. Registry → Energy Token (mint_tokens_direct)
   ┌────────────┐                    ┌────────────┐
   │  Registry  │ ──CPI──────────▶  │Energy Token│
   │            │   mint_tokens_    │            │
   │ Calculates │   direct()        │ Mints GRX  │
   │ settlement │                   │ to user    │
   └────────────┘                   └────────────┘

3. Trading → Token Program (transfers)
   ┌────────────┐                    ┌────────────┐
   │  Trading   │ ──CPI──────────▶  │   Token    │
   │            │   transfer()      │  Program   │
   │ Matches    │                   │            │
   │ orders     │                   │ Moves GRX  │
   └────────────┘                   └────────────┘
```

TOKEN ECONOMICS:
════════════════════════════════════════════════════════════════════════════════
• Minting:     1 GRX = 1 kWh of verified renewable energy
• Supply:      Dynamic (minted on energy production, burned on consumption)
